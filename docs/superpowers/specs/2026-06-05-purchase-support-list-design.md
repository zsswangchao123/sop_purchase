# 采购配套清单后台设计

## 背景

项目用于管理导入的蔬菜等主商品，并根据后台配置的配套规则，自动计算需要采购的辅料、包材或其他配套项，生成采购清单、明细价格和总价格。

第一版目标是把核心闭环跑通：商品配置、配套项配置、规则配置、手动或 Excel 导入商品、自动生成采购清单、修改本次单价、计算总价、状态留档和 Excel 导出。

## 范围

### 包含

- 商品管理：维护会被导入或选择的主商品。
- 配套项管理：维护需要采购的辅料、包材或其他配套物品。
- 配套规则管理：为每个商品配置一个或多个配套项规则。
- 采购清单生成：支持手动选择商品和 Excel 导入商品。
- 采购清单价格：使用配套项默认单价初始化本次单价，允许在清单中修改本次单价。
- 采购清单状态：支持草稿、已确认、已采购。
- 采购清单导出：支持导出 Excel。

### 不包含

- 供应商报价体系。
- 多级审批流。
- 库存扣减。
- 财务付款。
- 复杂权限体系。

这些能力可以作为后续扩展，但不进入第一版 MVP。

## 技术栈

- 后端：Java Spring Boot。
- 前端：Vue 3。
- 数据库：MySQL。
- Excel：后端负责模板下载、导入解析和导出生成。

## 用户和使用流程

后台用户先维护商品、配套项和配套规则。需要采购时，用户可以在页面手动选择商品并输入数量，也可以上传 Excel 商品清单。系统根据商品配套规则计算配套项采购数量，带出默认单价并生成草稿采购清单。

用户可在草稿中调整本次单价或备注。确认无误后，将清单状态改为已确认。采购完成后，将清单状态改为已采购。清单可随时查看和导出。

## 模块设计

### 商品管理

维护主商品信息。

字段：

- 商品编码。
- 商品名称。
- 规格。
- 单位。
- 备注。
- 启用状态。

商品编码用于 Excel 导入时优先匹配。商品名称可作为辅助匹配和页面展示。

### 配套项管理

维护需要采购的配套物品。

字段：

- 配套项编码。
- 配套项名称。
- 采购单位。
- 默认单价。
- 备注。
- 启用状态。

默认单价用于生成采购清单时初始化本次单价。历史清单保存价格快照，不受后续默认单价修改影响。

### 配套规则管理

为每个商品配置配套项规则。一个商品可以绑定多个配套项。

字段：

- 商品。
- 配套项。
- 计算方式。
- 基准商品数量。
- 配套项数量。
- 取整方式。
- 启用状态。

计算方式：

- 固定数量：只要本次清单包含该商品，就固定产生指定数量的配套项。
- 按比例：按商品数量和规则比例计算配套项数量。

取整方式：

- 向上取整：适合箱子、冰袋、标签等不可拆分物品。
- 保留小数：适合按 kg、米、卷等计量的物品，第一版保留 2 位小数。

### 采购清单生成

支持两种入口：

- 手动选择商品：搜索商品，输入数量，生成采购清单。
- Excel 导入：上传商品编码、商品名称、数量，系统匹配商品并生成采购清单。

生成后页面展示：

- 本次输入的商品清单。
- 自动计算出的配套项明细。
- 默认单价。
- 本次单价。
- 数量。
- 金额。
- 总金额。
- 未匹配商品、缺少规则或规则异常提示。

### 采购清单历史

支持查看历史清单和状态流转。

状态：

- 草稿：可修改本次单价和备注，可重新生成或删除。
- 已确认：锁定商品输入和配套项明细，不允许修改核心数据。
- 已采购：归档记录，只允许查看和导出。

第一版不做审批流。状态流转为草稿到已确认，已确认到已采购。

## 计算规则

固定数量：

```text
配套项数量 = support_quantity
```

比例计算：

```text
配套项数量 = 商品数量 / base_quantity * support_quantity
```

向上取整：

```text
最终数量 = ceil(配套项数量)
```

保留小数：

```text
最终数量 = round(配套项数量, 2)
```

金额：

```text
金额 = 最终数量 * 本次单价
总金额 = 所有配套项金额合计
```

如果多个商品算出同一个配套项，系统按配套项合并数量和金额，同时保存来源明细。

## Excel 导入

第一版模板字段：

- 商品编码。
- 商品名称。
- 数量。
- 备注。

匹配规则：

- 优先按商品编码匹配。
- 商品编码为空时，可按商品名称匹配。
- 名称匹配必须唯一，否则提示用户修正。

导入后生成草稿采购清单。存在未匹配商品或非法数量时，不保存清单，返回错误明细。

## 异常处理

- 商品编码不存在：导入失败并提示行号和商品编码。
- 商品名称匹配到多个商品：导入失败并提示用户使用商品编码。
- 商品停用：导入失败或手动选择时不可选。
- 商品没有启用配套规则：允许生成清单，但在结果中提示该商品无配套规则。
- 配套项停用：对应规则不参与计算，并提示规则异常。
- 数量为空、负数或非数字：导入失败并提示行号。
- 本次单价小于 0：禁止保存。
- 已确认或已采购清单：禁止修改商品输入、配套项数量和本次单价。

## 测试策略

后端测试重点覆盖采购计算逻辑：

- 固定数量规则能正确生成配套项。
- 比例规则能正确生成配套项。
- 向上取整能正确处理小数结果。
- 保留小数能保留 2 位。
- 多个商品生成同一个配套项时能正确合并。
- 合并后的配套项能保存来源明细。
- 修改本次单价后金额和总金额重算。
- 草稿、已确认、已采购状态的编辑限制正确。
- Excel 导入遇到未知商品、重复名称、非法数量时返回清晰错误。

前端测试重点覆盖关键交互：

- 商品、配套项、规则表单校验。
- 手动生成采购清单流程。
- Excel 导入错误提示展示。
- 草稿清单修改本次单价后金额更新。
- 已确认和已采购清单页面禁止编辑核心数据。
